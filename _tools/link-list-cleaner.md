## 왜 링크 추출 정리기가 필요할까요?
회의 메모, 채팅 로그, 공지 초안, 리서치 문서를 다루다 보면 링크가 본문 사이에 섞여 있어서 **필요한 URL만 따로 모으기 어렵다**는 문제가 자주 생깁니다.

예를 들어 이런 상황이 많습니다.

- 여러 문단 속에 참고 링크가 여기저기 흩어져 있을 때
- 같은 링크가 메신저 답장 과정에서 여러 번 중복될 때
- `utm_source`, `fbclid`, `msclkid` 같은 추적 파라미터가 붙어 공유용 링크가 지저분할 때
- `www.example.com`처럼 프로토콜이 빠진 링크까지 같이 모아야 할 때
- `[자료 링크](https://example.com)`처럼 마크다운 문서 안에 들어간 링크도 정리해야 할 때
- 도메인별로 묶어서 정리해 팀 문서에 넣고 싶을 때

이 도구는 본문 속 URL만 빠르게 뽑아 **중복을 줄이고, 추적 파라미터를 덜어내고, 공유하기 쉬운 링크 목록**으로 정리해주는 utility형 도구입니다. URL 목록, 마크다운 글머리 목록, HTML 링크 목록 중 필요한 형식으로 복사할 수 있고, 입력 내용은 서버로 보내지 않고 브라우저 안에서만 처리합니다.

## 어떻게 쓰나요?
1. 링크가 섞인 텍스트를 그대로 붙여넣습니다.
2. `중복 제거`, `추적 파라미터 삭제`, `도메인별 정렬` 옵션을 필요한 만큼 켭니다.
3. 복사용 출력 형식을 URL 목록, 마크다운 글머리 목록, HTML 링크 목록 중에서 고릅니다.
4. 결과 목록과 도메인 요약을 확인합니다.
5. 정리된 링크를 복사해 노션, 문서, 메신저, 참고자료 목록에 바로 붙여 넣습니다.

## 이런 상황에서 특히 유용해요
### 1) 회의 메모나 리서치 노트를 정리할 때
본문 전체를 다시 읽지 않아도 링크만 추출해 참고자료 섹션을 빠르게 만들 수 있습니다.

### 2) 메신저 대화에서 공유 링크를 정리할 때
같은 링크가 여러 번 등장해도 중복을 제거해서 깔끔한 목록으로 정리할 수 있습니다.

### 3) 외부 공유 전에 추적 파라미터를 줄이고 싶을 때
불필요한 `utm_*`, `fbclid`, `gclid`, `msclkid` 등을 덜어내 공유용 URL을 더 단정하게 만들 수 있습니다.

### 4) 마크다운·HTML 참고자료 목록을 만들 때
정리된 링크를 `- <https://...>` 형태의 마크다운 목록이나 `<a href="...">...</a>` 형태의 HTML 링크 목록으로 바로 복사할 수 있습니다.

## FAQ
### 링크가 한 줄에 여러 개 있어도 되나요?
네. 한 줄 안에 여러 URL이 있어도 순서대로 찾아 목록으로 정리합니다. `www.`로 시작하는 링크는 공유하기 쉽도록 `https://` 형태로 보정하고, 마크다운 링크 안의 URL도 함께 잡습니다.

### 추적 파라미터를 지우면 원래 링크가 망가지지 않나요?
일반적인 공유 링크에서는 대부분 문제 없습니다. 다만 일부 서비스는 특정 파라미터가 기능에 필요할 수 있으니 중요한 링크는 결과를 한 번 눌러 확인하는 편이 안전합니다.

### 링크를 도메인별로 묶는 이유가 뭔가요?
자료 출처를 빠르게 훑기 좋고, 같은 사이트 링크가 얼마나 많은지 바로 보이기 때문입니다. 리서치 자료 정리나 레퍼런스 정리에 특히 편합니다.
