## 개인정보 노출 점검기가 왜 필요할까요?
메신저 공지, 고객 안내문, 신청서 초안, 내부 문서 공유 같은 작업을 하다 보면 **복붙 과정에서 민감한 정보가 그대로 남는 실수**가 생각보다 자주 생깁니다.
특히 전화번호, 이메일, 주민등록번호 유사 문자열, 카드번호형 숫자열, 계좌번호처럼 **한 번 외부로 나가면 되돌리기 어려운 정보**는 발송 전에 빠르게 한 번 더 확인하는 습관이 중요합니다.

이 도구는 그런 실수를 줄이기 위해 만들었습니다.
텍스트를 붙여넣으면 **개인정보 노출 가능성이 있는 패턴을 바로 찾아주고, 마스킹 예시까지 함께 보여줍니다.**

## 이런 상황에서 특히 유용해요
### 1) 공지문이나 안내 메시지 발송 전 점검
단체 공지나 신청 안내문에 연락처, 링크, 계좌 정보가 섞여 있을 때 어떤 부분이 그대로 드러나는지 빠르게 확인할 수 있습니다.

### 2) 문서 공유 전 비식별 처리
보고서, 상담 기록, 회의 메모를 외부에 공유하기 전에 전화번호와 이메일 같은 직접 식별 정보를 먼저 가릴 수 있습니다.

### 3) 복붙한 텍스트 정리 중 마지막 안전 점검
여러 문단을 편집하면서 숫자열이 남아 있는지 놓치기 쉬울 때, 마스킹 결과를 보며 빠르게 재검토하기 좋습니다.

## 어떻게 동작하나요?
1. 점검할 텍스트를 붙여넣습니다.
2. 전화번호, 이메일, 주민등록/사업자/카드번호형 패턴, 계좌형 숫자열, 링크 유도 표현을 검사합니다.
3. 감지된 항목을 유형별로 보여줍니다.
4. 체크박스로 원하는 항목만 마스킹해 결과를 확인합니다.
5. 마스킹 결과를 복사해 안전한 공유용 초안으로 활용합니다.

## 사용 방법
1. 메시지, 문서, 공지문 초안을 입력합니다.
2. 어떤 항목을 마스킹할지 체크합니다.
3. 감지 목록에서 어떤 패턴이 잡혔는지 확인합니다.
4. 아래 마스킹 결과를 검토합니다.
5. 필요하면 복사해서 외부 공유용 문서에 붙여넣습니다.

## 사용 예시
### 예시 1) 고객 안내 메시지 점검
- 입력 내용: 담당자 번호, 회신 메일, 문의 링크가 섞인 안내문
- 확인 포인트: 전화번호와 이메일이 그대로 노출되는지

→ 발송 전에 직접 식별 정보를 가린 버전을 빠르게 만들 수 있습니다.

### 예시 2) 상담 기록 공유 전 비식별 처리
- 입력 내용: 상담 메모 일부, 연락처, 계좌번호 유사 문자열 포함
- 확인 포인트: 숫자열이 민감정보처럼 보일 수 있는지

→ 민감한 숫자 패턴을 다시 확인하고 안전한 공유본 초안을 만들기 좋습니다.

## FAQ
### 입력 내용은 안전한가요?
브라우저 내부에서만 처리되도록 만들었기 때문에 서버 전송 없이 바로 점검할 수 있습니다. 다만 공용 PC에서는 브라우저 자동완성이나 클립보드 사용에 주의하세요.

### 감지 결과가 많으면 모두 삭제해야 하나요?
꼭 그렇지는 않습니다. 공개가 필요한 대표 연락처나 공식 링크도 있을 수 있으니, 어떤 정보가 외부 공개용인지 먼저 구분한 뒤 필요한 부분만 마스킹하면 됩니다.

### 정규식 기반이면 놓치는 경우도 있나요?
있습니다. 사람 이름, 상세 주소, 사내 프로젝트 코드처럼 문맥 의존적인 민감정보는 패턴만으로 완벽히 찾기 어렵습니다. 이 도구는 1차 안전 점검용으로 쓰는 것이 가장 좋습니다.

## 요약
개인정보 노출 점검기는 **문서와 메시지 속 민감정보 패턴을 빠르게 찾아 마스킹 예시까지 보여주는 checker형 도구**입니다.
외부 발송 전 마지막 안전 점검이 필요할 때 특히 유용합니다.
