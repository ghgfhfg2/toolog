## JSON 합치기, 언제 쓰면 좋나요?
**JSON 합치기 툴**은 같은 스키마로 분할 저장된 파일들을 빠르게 하나로 묶어야 할 때 유용합니다.

예: 로그 분할 파일, 배치 결과 파일, 페이지 단위 수집 JSON 등.

API 응답을 페이지별로 저장했거나, 여러 배치 결과를 하나의 `merged.json`으로 만들어야 할 때 파일을 다시 서버에 올리지 않고 로컬 브라우저에서 확인할 수 있습니다. 작은 응답 조각은 파일로 저장하지 않고 바로 붙여넣어 JSON 배열 합치기와 객체 병합 결과를 테스트할 수 있습니다.

## 핵심 기능
- JSON 파일 다중 업로드
- API 응답·로그 조각 직접 붙여넣기
- 자동 병합 모드(배열/객체 구조 자동 감지)
- 배열 이어붙이기/객체 키 병합/루트 배열 감싸기 수동 선택
- 객체 안의 공통 배열 자동 이어붙이기
- 결과 미리보기 + 결과 복사 + `merged.json` 다운로드
- 중복 배열 항목 제거 + 객체 키 충돌 개수 확인
- 잘못된 JSON, 수동 모드 구조 불일치, 과도한 파일 크기 안내
- 예시 데이터로 병합 방식 먼저 확인

## 사용 방법
1. JSON 파일 여러 개를 업로드합니다.
2. 파일이 없다면 JSON 텍스트를 붙여넣습니다. 여러 JSON 문서는 `---` 한 줄로 구분합니다.
3. 병합 방식을 선택합니다(기본: 자동).
4. **JSON 합치기** 버튼을 누릅니다.
5. 결과를 확인한 뒤 복사하거나 다운로드합니다.

## 주의사항
- 객체 병합 모드에서 동일 키 충돌 시 뒤 파일 값이 우선합니다.
- 수동 객체 병합 모드는 모든 파일의 루트가 객체여야 하며, 구조가 맞지 않으면 병합하지 않고 오류를 안내합니다.
- 배열 중복 제거는 항목의 키 순서를 정렬해 비교하므로 `{ "a": 1, "b": 2 }`와 `{ "b": 2, "a": 1 }`처럼 같은 내용의 객체를 중복으로 봅니다.
- 브라우저 메모리 보호를 위해 한 번에 최대 50개, 합계 20MB까지 처리합니다.
- 붙여넣기 입력과 업로드 파일을 함께 사용할 때도 20MB 합계 제한을 적용합니다.
- 업로드한 파일과 결과는 서버로 전송되지 않고 현재 브라우저 안에서만 처리됩니다.
