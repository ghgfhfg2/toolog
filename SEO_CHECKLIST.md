# SEO 체크리스트 (Toolog)

## 1) 배포/인덱싱 기본
- [ ] `https://toolog.sooyadev.com/sitemap.xml` 접근 가능
- [ ] `https://toolog.sooyadev.com/robots.txt` 접근 가능
- [ ] Google Search Console 사이트 등록/소유권 확인
- [ ] Search Console에 sitemap 제출

## 2) GSC 기반 우선순위 선정
매주 먼저 확인:
- [ ] 최근 28일 검색어 노출 상위 20개 확인
- [ ] 최근 28일 페이지 노출 상위 20개 확인
- [ ] 평균 순위 8~20위 후보 페이지 표시
- [ ] 노출은 있으나 CTR이 낮은 후보 페이지 표시
- [ ] 후보 페이지별 실제 검색어 표현 3~5개 기록

우선순위:
1. 노출 있음 + 평균 순위 8~20위
2. 노출 있음 + CTR 낮음
3. 색인됨 + 검색 의도는 크지만 노출 낮음
4. 새 툴 또는 검색 수요가 불확실한 페이지

## 3) 페이지별 온페이지 점검
GSC 후보 페이지부터 확인:
- [ ] 제목(title) 앞부분에 메인 키워드 포함
- [ ] 메타 설명(description) 70~110자 내 핵심 가치 포함
- [ ] H1과 첫 문단에 사용자가 검색한 표현 자연스럽게 반영
- [ ] H2/H3 구조로 문제→사용법→예시→요약 구성
- [ ] 내부 링크 2개 이상 연결
- [ ] 외부 신뢰 출처 1개 이상 연결
- [ ] FAQ 2~3개 이상 + FAQ 스키마 노출

## 4) 클러스터/내부 링크 점검
- [ ] 홈에서 대표 검색 수요 묶음으로 연결
- [ ] 이미지, 텍스트, 생활/금융 계산 같은 클러스터가 명확함
- [ ] 관련 도구 앵커 문구가 실제 검색 표현과 가까움
- [ ] 유망 페이지가 홈/카테고리에서 2클릭 이내 노출됨

## 5) 기술 SEO 점검
- [ ] SoftwareApplication JSON-LD 정상 출력
- [ ] BreadcrumbList JSON-LD 정상 출력
- [ ] FAQPage JSON-LD(해당 페이지) 정상 출력
- [ ] 썸네일 이미지 alt 텍스트 일관성 유지
- [ ] 모바일 가독성(폰트/간격/터치 타겟) 점검

## 6) 성능/UX 점검
- [ ] 이미지 용량 최적화(WebP/JPEG 우선)
- [ ] Core Web Vitals 경고 페이지 확인
- [ ] 첫 문단 3초 이해 가능한 결론형 문장 적용
- [ ] 관련 도구(related tools)로 내부 순환 동선 확인

## 7) 운영 루틴
- [ ] 발행 후 24~72시간 내 Search Console 색인 상태 확인
- [ ] 수정 후 7~14일 뒤 같은 검색어의 노출·순위·CTR 비교
- [ ] 노출 증가 없이 정체된 페이지는 제목/H1/내부 링크 재점검
- [ ] 순위는 올랐지만 CTR 낮은 페이지는 제목/메타 재작성
- [ ] 월 1회 키워드 리프레시(검색량/의도/GSC 실적 기준)
