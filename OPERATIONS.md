# Toolog 운영 핸드북 v1

## 기술 스펙
- 플랫폼: Jekyll + GitHub Pages
- 배포: Git 기반 자동 배포
- 운영 방식: 초안 작성 후 승인 기반 발행

## 운영 목표
- 대중이 웹에서 바로 쓸 수 있는 실용 툴 콘텐츠를 지속 업데이트
- SEO 최적화를 통해 재방문/체류시간 증가
- 누적 트래픽 기반 애드센스 수익 향상

## 발행 리듬
- 하루 7회 사이클: 10, 12, 14, 16, 18, 20, 22시
- 사이클 절차
  1) 주제 선정
  2) 초안 작성 (`_drafts`)
  3) 검수 요청
  4) 승인 시 발행 (`_posts` 이동)

## 작성 기준
- 톤: 정확한 정보 전달 중심
- 주제: 일반 사용자도 바로 쓰는 실용 웹툴/문서
- 구조: 문제 → 설명 → 예시 → 요약
- 제목: 과장/낚시 지양, 검색의도-본문 일치

## 승인/배포 정책
- 자동 발행 금지
- 승인 전에는 `_drafts` 유지
- 승인 시 `scripts/publish_draft.sh`로 `_posts` 전환 후 커밋/푸시
- 반려 시 수정 또는 주제 재작성

## 품질 관리 체크리스트
- 기존 글과 주제 중복 최소화
- 내부 링크 점진적 축적
- 오탈자/링크/메타 정보 점검
- 주기 성과 보고 후 주제 전략 보정
